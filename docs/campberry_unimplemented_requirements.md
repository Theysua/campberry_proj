# Campberry 项目未完全落地需求清单

根据项目需求文档与当前代码库的对比分析，以下是尚未完全实现或仅部分实现（即作为前端UI Mockup存在）的需求清单。这些需求主要分为两大类：需要后端支持的核心功能，以及需要进一步完善的前端交互功能。

## 1. 核心后端与动态功能 (Backend & Dynamic Functionality)

这类需求是项目从当前静态原型（Prototype）转变为功能完备的动态应用所需实现的关键部分。它们的缺失是符合项目当前开发阶段的。

| ID | 需求名称 | 状态 | 缺失内容分析 |
| :--- | :--- | :--- | :--- |
| **FR-036** | Program Database (2100+ items) | **未实现** | 这是最核心的资产。目前项目使用的是一个包含50个项目的静态JSON文件作为模拟数据，尚未建立或接入包含2100+个项目的完整数据库。 |
| **FR-028** | Auth Redirect with Context | **未实现** | 应用目前没有实现路由守卫（Route Guards）。访问需要登录的页面（如`/my-lists`）不会自动跳转到登录页，这是完整用户认证流程中的关键一环。 |
| **FR-019** | Filter-URL Sync | **未实现** | 搜索页面的筛选状态与URL没有同步。刷新页面或分享URL后，当前的筛选条件会丢失，无法复现搜索结果。 |
| **FR-026** | Sign In Page | **部分实现** | 提供了完整的登录UI，但没有实际的认证逻辑。用户无法真正登录。 |
| **FR-027** | Sign Up Page | **部分实现** | 提供了完整的注册UI，但没有实际的用户创建逻辑。用户无法创建新账户。 |
| **FR-029** | My Lists Page | **部分实现** | 提供了“我的列表”页面的UI，包括创建列表的模态框，但所有操作都是前端模拟，没有后端数据交互。 |
| **FR-032** | Save / Bookmark Program | **部分实现** | “收藏”或“保存”按钮仅是UI元素，点击后没有实际功能，无法将项目保存到用户的个人列表中。 |
| **FR-034** | Add to List | **部分实现** | “添加到列表”功能目前仍主要是 UI 占位，缺少真实的列表选择、写入和持久化逻辑。 |
| **FR-037** | Guest Preview Limit / Registration Wall | **未实现** | 当前没有“游客最多查看 10 个活动，超过后需要注册”的会话计数和拦截逻辑，也没有保留原始跳转上下文。 |
| **FR-031** | Legal Pages | **未实现** | 尽管页脚有“服务条款”和“隐私政策”的链接，但对应的页面和路由（`/terms-of-service`, `/privacy-policy`）尚未创建。 |

## 2. 前端交互与功能完善 (UI & Feature Polish)

这类需求主要涉及前端UI的交互逻辑和一些细节功能的完善。在当前原型阶段，这些功能被简化或硬编码（Hardcoded）。

| ID | 需求名称 | 状态 | 缺失内容分析 |
| :--- | :--- | :--- | :--- |
| **FR-015** | Sort & Result Count | **部分实现** | 搜索结果页面显示了“排序：相关性”的按钮，但这是一个静态UI。当前缺少按截止日期排序以及升序/降序切换的真实逻辑。 |
| **FR-017** | Pagination | **部分实现** | 搜索结果页底部有分页的UI组件，但仅为静态展示，不支持点击翻页。 |
| **FR-005** | Homepage Hero & Search | **部分实现** | 首页的搜索框在输入关键词后直接跳转到`/search`，但没有将关键词作为查询参数（Query Parameter）传递过去。 |
| **FR-006** | Top Searches Tags | **部分实现** | 首页的“热门搜索”标签是静态按钮，点击后不会导航到带有预设筛选条件的搜索页面。 |
| **FR-011** | Filter Panel - Quick Filters | **部分实现** | 筛选面板中的所有筛选器（如Experts' Choice, Type, Season, Grade）目前都是UI占位符，点击后不会实时更新搜索结果。 |
| **FR-014** | Reset Filters & Filter Count | **部分实现** | “重置筛选”按钮没有功能，且筛选数量的显示是静态的“(3)”，不会根据用户的选择动态变化。 |
| **FR-016** | Program Card Component | **部分实现** | 卡片尚未稳定展示核心 B2B 信息组合：明确的 Deadline 字段、官方学校/机构网站入口，以及围绕顾问研究场景优化的信息层级。 |
| **FR-023** | Ratings & External Link | **部分实现** | 详情页尚未把“官方网站链接”作为核心 CTA 落实到真实数据和交互中；同时旧的 financial accessibility 方向已不再是优先范围。 |
| **FR-029A** | Research / Client List Reuse | **未实现** | 现有“我的列表”仅是 UI 占位，缺少真正的持久化、复用和面向顾问研究/客户展示的列表管理能力。 |
| **FR-033** | Share Program / List | **部分实现** | “分享”按钮是静态图标，点击后不会弹出带有“复制链接”或社交分享选项的浮窗（Popover）。 |
| **FR-003** | Ask Campberry CTA Banner & Modal | **部分实现** | CTA横幅被硬编码在搜索结果的第3个项目后显示，而不是按需求“每5个卡片插入一个”。点击按钮也不会弹出包含联系方式的模态框。 |

## 3. 新优先级说明

根据最新产品反馈，当前版本的主优先级已调整为以下 5 项：

1. Listings：搜索结果必须优先服务顾问筛选和浏览活动。
2. Deadlines：列表卡片与排序能力必须突出截止日期，并支持升序/降序。
3. Official Website Link：卡片或详情页必须可直达学校/机构官网。
4. Registration Wall：游客可查看 10 个活动，超过后要求注册，为未来收费做准备。
5. Saved Lists：注册顾问可保存和复用列表，用于研究和客户展示。

同时，以下方向降级或移出当前优先范围：

- “Free / Fully Funded” 相关筛选与定位
- “Impact on Admissions” 相关筛选与排序
- Compare 功能

这份清单可以作为后续开发工作的待办事项列表（To-do List），帮助开发团队明确下一步的开发重点。
