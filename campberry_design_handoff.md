## 一、Base44 设计风格分析总结

Base44 是一个面向 AI 网站构建的 SaaS 产品，其设计语言体现了当前最前沿的 Web 设计趋势。整体风格可以概括为 **"柔和现代主义"**（Soft Modernism），核心特征是大量使用柔和渐变色背景、超大圆角组件、胶囊形按钮、充足的留白以及精心设计的字重层次。这种风格传递出温暖、友好且专业的品牌调性，非常适合面向学生和家长群体的 Campberry 产品。

Base44 的视觉层次通过三种手段建立：首先是色彩对比——柔和的暖色渐变背景与深海军蓝的强对比区块交替出现；其次是字号跨度——从 79px 的主标题到 14px 的正文，形成了极为鲜明的视觉节奏；最后是空间节奏——区块之间使用大面积留白（通常 60-80px），而组件内部则保持紧凑。

## 二、前端标准设计语言规范（Design Tokens）

以下规范可直接作为 CSS 变量系统交接给下一个 agent 使用。

### 2.1 色彩体系

| Token 名称 | 值 | 用途说明 |
|---|---|---|
| `--primary` | `#011936` | 品牌主色，深海军蓝，用于标题、深色区块 |
| `--primary-light` | `#0a2f5c` | 主色浅变体，用于 hover 状态 |
| `--accent` | `#b8243f` | 强调色，品牌红，用于 CTA、重要标记 |
| `--accent-light` | `#d94f6a` | 强调色浅变体，用于 hover/focus |
| `--accent-gradient` | `linear-gradient(135deg, #b8243f, #e8506a)` | 强调色渐变，用于主按钮、徽章 |
| `--hero-gradient` | `linear-gradient(180deg, #dbeafe, #fce7d6, #fdf2e9)` | Hero 区域背景渐变（蓝→桃→暖白） |
| `--mint` | `#e0fff5` | 薄荷绿，用于功能卡片背景 |
| `--yellow` | `#fde047` | 亮黄色，用于 NEW 标签等 |
| `--orange` | `#ff8a3d` | 橙色，用于装饰性渐变 |
| `--bg` | `#f4f7fa` | 页面默认背景色 |
| `--surface` | `#ffffff` | 卡片/组件表面色 |
| `--text` | `#1e293b` | 主文本色 |
| `--text-secondary` | `#64748b` | 辅助文本色 |
| `--border` | `#e2e8f0` | 标准边框色 |
| `--border-light` | `#f1f5f9` | 浅边框/分隔线色 |

### 2.2 阴影体系

| Token 名称 | 值 | 用途 |
|---|---|---|
| `--shadow-sm` | `0 1px 3px rgba(0,0,0,0.04), 0 1px 2px rgba(0,0,0,0.06)` | 默认卡片阴影 |
| `--shadow-md` | `0 4px 12px rgba(0,0,0,0.06), 0 2px 4px rgba(0,0,0,0.04)` | hover 状态阴影 |
| `--shadow-lg` | `0 12px 24px -4px rgba(0,0,0,0.08), 0 4px 8px rgba(0,0,0,0.04)` | 浮起元素阴影 |
| `--shadow-xl` | `0 20px 40px -8px rgba(0,0,0,0.12)` | 最大层级阴影 |

### 2.3 圆角体系

| Token 名称 | 值 | 用途 |
|---|---|---|
| `--radius-sm` | `8px` | 小元素（输入框、小按钮） |
| `--radius-md` | `16px` | 中等元素（搜索框、表单） |
| `--radius-lg` | `24px` | 大元素（卡片、面板） |
| `--radius-pill` | `9999px` | 胶囊形（按钮、标签、徽章） |

### 2.4 字体与排版

**字体栈**: `'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`

| 元素 | 字号 | 字重 | 行高 | 字间距 |
|---|---|---|---|---|
| H1（页面主标题） | `clamp(32px, 5vw, 52px)` | 800 | 1.15 | `-0.03em` |
| H2（区块标题） | `28px` | 800 | 1.3 | `-0.03em` |
| H3（卡片标题） | `17-20px` | 700 | 1.4 | `-0.02em` |
| 正文 | `15px` | 400 | 1.7 | normal |
| 按钮文字 | `14-16px` | 600-700 | 1 | `-0.01em` |
| 标签/徽章 | `10-12px` | 600-700 | 1 | `0.03em` |

### 2.5 组件规范

**按钮（Buttons）**：所有按钮采用胶囊形（`border-radius: 9999px`）。主按钮使用 `accent-gradient` 背景并带有品牌色阴影；深色按钮使用 `primary` 背景；轮廓按钮使用透明背景 + 边框。所有按钮 hover 时上移 2px 并增强阴影。

**卡片（Cards）**：统一使用 `border-radius: 24px`，默认带 `shadow-sm`，hover 时过渡到 `shadow-lg` 并上移 4-6px。顶部可带 4px 的渐变色条作为类别标识。

**搜索框（Search）**：大圆角容器（`16px`），focus 时边框变为 accent 色并带有外发光效果（`box-shadow: 0 0 0 4px rgba(accent, 0.08)`）。

**标签（Tags）**：胶囊形，浅灰背景，hover 时背景加深。

**徽章（Badges）**：胶囊形，使用渐变背景，全大写字母，加宽字间距。

### 2.6 动画规范

**入场动画**: `fadeInUp`，从下方 24px 处淡入上移，持续 0.6s，使用 `cubic-bezier(0.16, 1, 0.3, 1)` 缓动函数。支持 `.delay-1` 到 `.delay-5` 的交错延迟（每级 0.1s）。

**交互动画**: 所有可交互元素使用 `transition: all 0.25s cubic-bezier(0.16, 1, 0.3, 1)`，hover 时 `translateY(-2px)` 到 `translateY(-6px)` 不等。

## 四、交接注意事项

下一个 agent 在基于此设计语言构建正式网站时，应注意以下几点：

**CSS 变量优先**：所有颜色、圆角、阴影值必须通过 CSS 变量引用，不要硬编码。这确保了全局一致性和主题切换能力。

**动画克制**：入场动画仅用于首屏 Hero 区域和关键 CTA 区块。过度使用动画会降低专业感。

**渐变使用规则**：渐变色仅用于三个场景——Hero 背景、主 CTA 按钮、卡片顶部装饰条。其余区域保持纯色。

**字重对比**：标题必须使用 700-800 字重，正文使用 400。中间字重（500-600）仅用于按钮和标签。避免使用 300 以下的字重。

**间距基准**：所有间距基于 8px 网格系统。常用值为 8/16/24/32/40/48/60/80px。
